# Claude Code: Правила работы с Kaiten Landing Factory

Этот файл определяет, как Claude должен создавать лендинги Kaiten в этом репозитории.

## 🎯 Основной принцип

**Один промпт → один готовый лендинг без правок**

Цель — создавать настолько качественные лендинги, чтобы ты мог отправить их разработчику сразу в публикацию, без цикла «напиши → проверь → переделай → снова проверь».

## 📌 Когда я создаю лендинг Kaiten

**Входная команда выглядит так:**

```
Создай лендинг Kaiten для [СФЕРА]

[опциональные детали]
```

**На выходе я должен дать:**

1. **Исследование целевой аудитории** — 2-3 абзаца о том, почему именно эта сфера нуждается в Kaiten
2. **Структура лендинга** — навигация по всем блокам
3. **Готовые тексты** — все разделы полностью написаны (заголовки, подзаголовки, описания, CTA)
4. **HTML-верстка** — полностью адаптивная, готовая к публикации
5. **SEO-данные** — title, description, keywords, Schema.org
6. **Интеграция кейсов** — реальные примеры клиентов (Эксмо, METRO, Строительная компания и т.д.)
7. **Чек-лист** — для финальной проверки перед публикацией

## 📝 Основные правила написания текстов

### Брендинг и голос Kaiten

- **Тон**: коллега, который разбирается в своём деле (умный, но не занудный)
- **Уверенность, но не дерзость**
- **Простота, но экспертность**
- **Никогда не лебезить**

Примеры ❌ НЕПРАВИЛЬНО:

- «Kaiten — это замечательный инструмент, который перевернёт вашу жизнь!»
- «Мы — лучшие на рынке и все это знают»
- «Вам просто необходимо попробовать Kaiten»

Примеры ✅ ПРАВИЛЬНО:

- «Kaiten помогает управлять проектами без потери видимости»
- «Вот что получат твои люди, если переключатся на Kaiten: [конкретный результат]»
- «Мы видели, как это работает на практике, вот примеры»

### Русский язык и англицизмы

- **Используй русские слова**, где есть аналоги: «доска» вместо «board», «управление» вместо «management»
- **Исключение**: канбан, скрам, бэклог, спринт, стейкхолдер — допускаются, но только если раньше в тексте не объяснял
- **Первый раз термин**: пиши с раскрытием в скобках, например: «CRM-система (управление отношениями с клиентами)» — первый раз, потом просто CRM
- **Никаких французких и английских фраз** типа «добавлять ценность», «синергия», «инновационный подход»

### Структура текста

- **Краткость**: уважаем время читателя, убираем лишнее
- **Понятность**: сложные идеи разбиваем на простые
- **Полезность**: каждый абзац отвечает на вопрос «как это поможет читателю?»
- **Примеры**: опираемся на реальные кейсы, а не абстрактные высказывания

## 🏗️ Структура лендинга (на основе реальных эталонов)

**ВАЖНО:** Структура копируется из `landing_tz_examples/`. Открой эталон похожей сферы и следуй его структуре и тону.

Типичная структура (может варьироваться по сферам):

1. **Заголовок в таблице** — название решения
   ```
   | Название лендинга |
   | :---- |
   ```

2. **Hero блок** — заголовок + подзаголовок
   - Заголовок: конкретное решение для конкретной аудитории (3-5 слов)
   - Подзаголовок: выгода (2 предложения)
   - CTA кнопка: "Попробовать бесплатно"

3. **Кейсы** (3 реальных компании в таблице)
   - Название компании + цитата с результатом + ссылка на кейс
   - Без логотипов (или с описанием где они в макете)

4. **Основной УТП блок** — заголовок + выгоды (3-4 подблока)
   - Заголовок: почему эта сфера нуждается в Kaiten
   - 3-4 практические выгоды с кратким описанием

5. **Блоки по функциям** (2-4 блока)
   - Для каждой функции: название + как работает + визуал (описание)
   - Таблица 2 колонки: описание слева, визуал справа

6. **Блоки для разных ролей** (4-6 ролей/отделов)
   - Таблица со всеми ролями и их возможностями
   - Описание того как каждая роль работает в Kaiten

7. **Документы и доп. функции** (опционально)
   - Блок про встроенные документы, AI, интеграции и т.д.

8. **Вспомогательные блоки** (опционально)
   - В зависимости от сферы: графики, декомпозиция, отчеты и т.д.

9. **Онбординг блок** — "Начните за 30 минут"
   - 3-4 шага с кратким описанием
   - CTA: кнопка на регистрацию

10. **FAQ блок** — "Часто спрашиваемые вопросы"
    - 5-7 вопросов по сфере
    - Короткие ответы (2-3 предложения)
    - Используй `##` для вопросов

## 🎨 Дизайн и верстка

### HTML и CSS

- **Семантический HTML5**: `<header>`, `<section>`, `<article>`, `<footer>`, не просто `<div>`
- **Таблицы в Markdown → HTML таблицы**: эталоны используют таблицы Markdown, я конвертирую их в `<table>` с правильной разметкой
- **Адаптивность**: mobile-first подход, тестирую на мобильных устройствах
- **Flexbox/Grid**: современный подход к лейауту
- **CSS переменные**: если используется система стилей Kaiten

### Структура из эталонов → HTML

1. **Таблицы в Markdown** → `<table>` с `<tr>`, `<td>`
2. **Заголовки `# **текст**`** → `<h1>`, `<h2>`, `<h3>` с правильным семантическим уровнем
3. **Списки по ролям/функциям** → либо таблица, либо `<ul>`/`<ol>`
4. **Визуалы (`![][image1]`)** → `<img src="path" alt="description">`

### Стили Kaiten

- **Основные цвета**: проверяю в `references/brand_policy/`
- **Типография**: без прикрас, читаемо
- **Таблицы**: border-collapse, читаемые ячейки, хороший padding
- **Расстояния**: consistent padding/margin между блоками
- **Тёмная тема**: если требует дизайн-система

### Изображения и визуалы

- **Alt-текст обязателен** для каждого изображения
- **В эталонах**: указан как `![][image1]` → я подставляю реальные пути или placeholder'ы
- **Оптимизация**: если используются растровые — указываю формат (webp приоритет, fallback jpg)
- **Высокое качество**: не использую низкокачественные скриншоты

## 🔍 SEO и метаданные

### URL

- **Формат**: `kaiten-dlya-kanban-komand` (русские буквы, если поддерживает сервер, иначе транслитерация)
- **Описательный**: содержит ключевое слово целевой страницы

### Meta tags

```html
<title>Kaiten для канбан-команд</title>
<meta name="description" content="Доска для управления задачами в канбан-методе. Видимость всех процессов, контроль спринтов и быстрая доставка.">
<meta name="keywords" content="kanban, доска, управление проектами, спринт">
```

- **Title**: 50-60 символов включая ключевое слово
- **Description**: 150-160 символов, призыв к действию
- **Keywords**: основной ключ + 2-3 варианта

### Schema.org

- **Organization schema** для контактной информации
- **BreadcrumbList** для навигации
- **Product schema** (если нужен) для Kaiten как продукта
- **FAQPage schema** для FAQ блока

- [ ] **Alt-тексты** для всех изображений

- [ ] **Орфография и пунктуация** безупречны

- [ ] **Тон голоса** соответствует стилю Kaiten (не помпезный, не занудный)
