# ПОЛНЫЙ ЦИКЛ: От промпта к готовому лендингу

Это инструкция как я работаю: от анализа целевой аудитории до финального дизайна и верстки лендинга.

---

## 🔄 ПРОЦЕСС РАБОТЫ

### ЭТАП 1️⃣: АНАЛИЗ ЦА (В ЧАТ, БЕЗ ФАЙЛОВ)

**Что я делаю (1-2 часа):**
- Примеряю на себя роль руководителя этой команды
- Анализирую боли этой сферы (форумы, отзывы, конкуренты)
- Выписываю 3-5 основных болей
- Определяю какие фичи Kaiten их закрывают
- **Ищу кейсы из официальной библиотеки** https://kaiten.ru/blog/tag/case/

**Я пишу в чат:**
- 2-3 абзаца про целевую аудиторию (кто это, что они делают)
- 3-5 конкретных болей (на языке ЦА, не на языке Kaiten)
- Какие фичи Kaiten решают эти боли
- Какие реальные кейсы найду в библиотеке (компания, результат)

**ВАЖНО:** 
- ❌ БЕЗ файлов! Просто текст в чате
- ❌ БЕЗ придуманных примеров! Только реальные кейсы из https://kaiten.ru/blog/tag/case/
- ✅ На языке ЦА (не на языке Kaiten)

---

### ЭТАП 2️⃣: ОТДЕЛЬНЫЙ ФАЙЛ ТЗ СО СТРУКТУРОЙ И ПОЛНЫМИ ТЕКСТАМИ

**Что я делаю (3-4 часа):**

1. **Выбираю эталон** из [`landing_tz_examples`](https://github.com/katyakaiten/katyakaiten/tree/main/kaiten-landing-factory/references/landing_tz_examples) похожей сфере
   - Пример: для agile-команд → ищу IT-лендинг или маркетинг
   - Копирую СТРУКТУРУ (блоки, порядок, макет)
   - Копирую ТОН (как пишется, какой язык)

2. **Пишу полное ТЗ** для дизайнера (НЕ анализ, а готовый контент):
   - **Hero блок**: точный заголовок + подзаголовок
   - **Кейсы**: реальные из https://kaiten.ru/blog/tag/case/ (компания, цитата, ссылка)
   - **Каждый блок**: полный готовый текст (не заглушки)
   - **Визуалы**: подробное описание что там должно быть
   - **CTA**: точные тексты кнопок
   - **SEO**: title, description, keywords

3. **ВАЖНО при кейсах:**
   - ❌ НЕ придумываю кейсы
   - ✅ ТОЛЬКО из реальной библиотеки https://kaiten.ru/blog/tag/case/
   - ✅ Выбираю 2-3 кейса которые релевантны ЦА
   - ✅ Копирую цитату, компанию, ссылку прямо из кейса

**На выходе — ОТДЕЛЬНЫЙ ФАЙЛ:**
- `tz_[название].md` — полное ТЗ для дизайнера
  - ❌ НЕ содержит анализ ЦА (это остался в чате)
  - ✅ ТОЛЬКО структура блоков + полные готовые тексты
  - ✅ Заголовки, подзаголовки, описания — точные, как для публикации
  - ✅ Реальные кейсы с ссылками
  - ✅ Описание всех визуалов
  - ✅ SEO-метаданные

**Структура файла:**
```markdown
# ТЗ: Кайтен для [название]

## 1. Hero блок
### Заголовок
[Точный текст заголовка]

### Подзаголовок
[Точный текст подзаголовка]

### CTA
- [Кнопка 1]
- [Кнопка 2]

### Визуал
[Подробное описание что должно быть]

---

## 2. Кейсы (из реальной библиотеки)
| Компания | Цитата | Ссылка |
| :---- | :---- | :---- |
| Компания 1 | "Реальная цитата из кейса" | [Читать кейс](ссылка) |
| Компания 2 | ... | ... |

---

## 3. Основные выгоды
### Выгода 1: [название]
[Полный текст, готовый к публикации]

### Визуал
[Описание]

...
```

### ЭТАП 3️⃣: ПРАВКИ (ОПЦИОНАЛЬНО, В ЧАТ)

**Если нужны правки:**
- Ты пишешь в чат: "Переделай текст в Hero блоке", "Добавь еще один кейс", "Измени FAQ"
- Я обновляю ТОЛЬКО то, что просила
- Отправляю обновленный файл

**На выходе:**
- Финальное ТЗ, согласованное с тобой

---

### ЭТАП 4️⃣: ДИЗАЙН (ОПЦИОНАЛЬНО)

**Если нужны макеты:**
- Беру финальное ТЗ
- Создаю визуальные макеты (описания или мокапы)
- Для каждого блока: раскладываю тексты из ТЗ + визуалы + требования к дизайну
- Адаптирую для mobile/tablet/desktop

**На выходе:**
- Описание макетов или макеты в Figma
- Все размеры и цвета определены
- Все взаимодействия описаны

---

### ЭТАП 5️⃣: ВЕРСТКА (ОПЦИОНАЛЬНО)

**Если нужен HTML:**
- Беру макеты/ТЗ
- Верстаю HTML5 + CSS
- Адаптивный дизайн (mobile-first)
- Встроенный CSS
- Работающий FAQ, кнопки и т.д.

**На выходе:**
- `landing.html` — готовая верстка
- `seo.json` — SEO-данные
- `checklist.md` — чек-лист перед публикацией

## 💡 ПРИМЕР РАБОТЫ

### Твой промпт:
```
Создай ТЗ для лендинга Kaiten редакций

Целевая аудитория: главные редакторы и редакторы в издательствах
Ключевые проблемы:
- Статьи теряются в чатах и гугл-доках
- Сложно отследить этапы: написание → правка → публикация
- Нет видимости кто над чем работает
```

### Мой процесс:

**Этап 1 (анализ в чат):**
```
Главные редакторы в издательствах борются с хаосом:
- Статьи хранятся в Google Docs, правки в Telegram, согласования в почте
- Никто не видит статус статьи (в работе? на правке? готова?)
- Часто теряются дедлайны и версии

Kaiten решает это:
- Все в одном месте (статья, правки, статус)
- Видимость по этапам (написание → редактура → публикация)
- Автоматические уведомления о дедлайнах
```

**Этап 2 (файл с ТЗ):**
Я создаю файл `tz_redakcii.md` с полной структурой и текстами:
```markdown
# Управляйте редакцией в Kaiten: от идеи до публикации

## Hero блок
### Заголовок
Все статьи, авторы и статусы — в одной системе. Видно кто пишет, кто редактирует, когда будет готово.

### CTA
Попробовать бесплатно

## Кейсы
| Хабр | "Кайтен закрыл все потребности от распределения задач до прозрачности процессов" | [Читать кейс] |
...

## Основные выгоды
### Видно статус каждой статьи
...
```

**Этап 3 (правки в чат):**
```
Ты: "Переделай текст про видимость статей, добавь про AI-ассистента для интервью"
Я: Обновляю файл и отправляю новую версию
```

**Этап 4 (макеты если нужны):**
Создаю описание визуалов для дизайнера

**Этап 5 (верстка если нужна):**
Верстаю HTML + SEO + чек-лист
