# ПОЛНЫЙ ЦИКЛ: От промпта к готовому лендингу

Это инструкция как я работаю: от анализа целевой аудитории до финального дизайна и верстки лендинга.

---

## 🔄 ПЯТЬ ЭТАПОВ РАБОТЫ

### ЭТАП 1️⃣: АНАЛИЗ ЦЕЛЕВОЙ АУДИТОРИИ (1-2 часа)

**Что я делаю:**
- Примеряю на себя роль руководителя этой команды
- Задаю себе вопросы: что мешает в работе, на что тратится время, что критично
- Анализирую боли этой сферы (открытые источники, форумы, отзывы)
- Смотрю как о них говорят конкуренты
- Выписываю 3-5 основных болей

**На выходе:**
- Четкое описание ЦА (кто руководитель этой команды)
- 3-5 конкретных болей (не абстрактно, а по-человечески)
- Какие фичи Kaiten закрывают эти боли

---

### ЭТАП 2️⃣: СОЗДАНИЕ ПОЛНОГО ТЗ (3-4 часа)

**Что я делаю:**
1. **Смотрю эталон-рефы** из папки [`landing_tz_examples`](https://github.com/katyakaiten/katyakaiten/tree/main/kaiten-landing-factory/references/landing_tz_examples):
   - Кайтен для веб-агентств.md
   - Кайтен для маркетинга.md
   - Кайтен для редакций.md
   - Посадочная страница для ИТ команды.md
   - Посадочная страница для Строительство, инженеры.md
   - Управление портфелем проектов.md
   - Госкомпании и учреждения.md

2. **Копирую структуру и тон** из подходящего эталона или **создаю новую** (если другая сфера)

3. **Пишу полное ТЗ** с:
   - Готовыми текстами для каждого блока
   - Описанием визуалов
   - Требованиями к дизайну
   - Чек-листом

**На выходе:**
- **ПОЛНОЕ ТЗ** (структурировано как эталоны в [`landing_tz_examples`](https://github.com/katyakaiten/katyakaiten/tree/main/kaiten-landing-factory/references/landing_tz_examples))
- Все тексты уже написаны
- Все визуалы описаны
- Готово для дизайнера

---

### ЭТАП 3️⃣: ОТРАБОТКА ПРАВОК (1-2 часа)

**Что я делаю:**
- Ты даёшь обратную связь: "текст в блоке X нужно переписать", "добавь ещё один кейс", "измени структуру"
- Я быстро правлю ТЗ
- Не переделываю весь лендинг, правлю только то что просила

**На выходе:**
- Финальное ТЗ, согласованное с тобой

---

### ЭТАП 4️⃣: ДИЗАЙН (2-3 часа)

**Что я делаю:**
- Беру финальное ТЗ
- Создаю макеты/мокапы в Figma/Adobe XD/Sketch (или описываю макеты)
- Для каждого блока:
  - Раскладываю тексты из ТЗ
  - Добавляю визуалы (скриншоты, иллюстрации, иконки)
  - Соблюдаю требования к цветам, шрифтам, отступам
  - Адаптирую для mobile/tablet/desktop

**На выходе:**
- Готовые макеты для разработчика
- Все размеры и цвета определены
- Все взаимодействия (кнопки, FAQ) описаны

---

### ЭТАП 5️⃣: ВЕРСТКА (2-3 часа)

**Что я делаю:**
- Беру готовые макеты
- Верстаю HTML/CSS по ним
- Адаптивная верстка (mobile-first)
- Встроенный CSS
- Семантический HTML5
- Работающий FAQ (раскрывается/скрывается)
- Все кнопки работают

**На выходе:**
- **ГОТОВЫЙ ЛЕНДИНГ**
- HTML и CSS
- Адаптивный
- Готов к публикации

---

## 📋 ЧТО МНЕ НУЖНО ОТ ТЕБЯ

Когда ты даёшь промпт, дай мне:

1. **Название сферы/аудитории**
   - Пример: "Маркетинг", "Веб-агентства", "Бухгалтерия"

2. **Примеры для вдохновения** (опционально)
   - Ссылки на готовые лендинги
   - Примеры ТЗ которые мне нужно использовать

3. **Кейсы** (если знаешь)
   - Какие компании использую эту сфер
   - Лучше 3-5 реальных компаний

4. **Особенности этой сфере** (если есть)
   - Что здесь критично
   - Что вызывает боль

**Пример промпта:**
```
Создай полный цикл лендинга Kaiten для [СФЕРА]

Ссылки на примеры:
- [ССЫЛКА НА ПОХОЖИЙ ЛЕНДИНГ]

Кейсы (если знаешь):
- [КОМПАНИЯ 1]
- [КОМПАНИЯ 2]

Особенности:
- [ЧТО ВАЖНО]
```

## 💡 ПРИМЕР РАБОТЫ

### Твой промпт:
```
Создай полный цикл лендинга Kaiten для маркетинговых отделов
```

### Мой процесс:

**Этап 1 (Анализ):**
- Я примеряю себя на PM маркетинга в крупной компании
- Задаю вопросы: что мешает управлять проектами, как сейчас работаем, на что уходит время
- Выписываю боли: хаос в задачах, потери в чатах, нет видимости по проектам

**Этап 2 (ТЗ):**
- Беру эталон "Кайтен для маркетинга.md" из папки [`landing_tz_examples`](https://github.com/katyakaiten/katyakaiten/tree/main/kaiten-landing-factory/references/landing_tz_examples)
- Копирую структуру блоков и тон
- Пишу полное ТЗ с текстами для всех блоков (как в эталоне)
- Подбираю кейсы (Бомбора, Darvin Studio, Out of Cloud и т.д.)
- Описываю все визуалы по образцу эталона

**Этап 3 (Правки):**
- Ты говоришь: "текст в блоке про документы нужно переписать"
- Я правлю только этот блок
- Отправляю исправленное ТЗ

**Этап 4 (Дизайн):**
- Беру финальное ТЗ
- Создаю макеты для каждого блока
- Раскладываю тексты, добавляю визуалы
- Соблюдаю палитру, шрифты, отступы

**Этап 5 (Верстка):**
- Верстаю по макетам
- HTML + CSS
- Адаптивный дизайн
- Готовый лендинг!
