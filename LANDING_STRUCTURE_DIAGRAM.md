# Структура лендинга Kaiten: Полная схема

## 📊 Визуальная структура

```
┌─────────────────────────────────────────────────────────────┐
│                     1. HERO БЛОК                            │
│  [Сфера] выбирают Kaiten                                   │
│  [Конкретная выгода] в одной системе                        │
│  [CTA: Попробовать бесплатно]                             │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│                    2. БЛОК КЕЙСОВ                           │
│  Реальные примеры других компаний (4-5 кейсов)             │
│  - Логотип + цитата + результат                            │
│  - Фамилия, должность, компания                            │
│  - Ссылка на полный кейс                                   │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│            3. БЛОКИ ЗАКРЫТИЯ РЕШЕНИЙ (3-4)                │
│  ❌ НЕ описываем текущие боли                               │
│  ✅ Показываем что получит человек в Kaiten               │
│                                                             │
│  Пример:                                                    │
│  ## Видишь весь прогресс на одной доске                   │
│  [Как это работает в Kaiten, конкретные шаги]             │
│  [Результат и выгода]                                      │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│         4. ГИБКИЕ БЛОКИ ПО СФЕРАМ (выбери 3-4)           │
│                                                             │
│  Для ИТ:                Для редакции:                      │
│  □ Документы            □ Документы                         │
│  □ Примеры команд       □ Примеры команд                   │
│  □ Чат & видео          □ Чат & видео                      │
│  □ Графики              □ Графики                          │
│  □ AI помощник          □ AI помощник                      │
│  □ Декомпозиция         □ Декомпозиция                     │
│                                                             │
│  Для веб-агентства:     Для юристов:                       │
│  □ Графики              □ Документы                         │
│  □ Примеры команд       □ Безопасность                     │
│  □ AI помощник          □ Интеграции                       │
│  □ Интеграции           □ На-premise опция                 │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│          5. БЛОК ПЕРЕНОС ДАННЫХ (ОБЯЗАТЕЛЬНЫЙ)            │
│  Экспорт из Jira, Trello, Notion                           │
│  Помогаем с миграцией, никаких потерь                      │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│        6. БЛОК БЫСТРЫЙ СТАРТ (ОБЯЗАТЕЛЬНЫЙ)              │
│  День 1, День 2, День 3 — что происходит                   │
│  Легко начать, быстро разбираются, интуитивно             │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│      7. БЛОК ГОТОВЫЕ ШАБЛОНЫ (ОБЯЗАТЕЛЬНЫЙ)              │
│  Готовые шаблоны пространств для этой сферы               │
│  Не нужно ничего настраивать, используй и работай         │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│     8. ЯРКИЙ СТАРТОВЫЙ БЛОК (ОБЯЗАТЕЛЬНЫЙ)               │
│  [Сильный результат]                                       │
│  [Конкретная выгода с цифрой]                             │
│  [CTA: Попробовать бесплатно]                             │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│           9. БЛОК FAQ (ОБЯЗАТЕЛЬНЫЙ)                       │
│  6-8 вопросов:                                             │
│  - Универсальные (цена, внедрение, безопасность)          │
│  - Специфические для сферы                                │
│  Короткие, конкретные ответы                              │
└─────────────────────────────────────────────────────────────┘
                           ↓
                    [FOOTER]
```

## 📋 Полный список блоков с пометками

| № | Блок | Обязательно? | Адаптируется? | Примечания |
|---|------|-------------|---------------|-----------|
| 1 | Hero | ✅ Да | ✅ Да | Одна для всех, формула одна |
| 2 | Кейсы | ✅ Да | ✅ Да | 4-5 кейсов, подбираешь по сфере |
| 3 | Решения (3-4) | ✅ Да | ✅ Да | Показываем результаты, не боли |
| 4 | Гибкие блоки (3-4) | ✅ Да | ✅✅ Очень | Выбираешь по сфере и аудитории |
| 5 | Перенос данных | ✅ Да | ⚪ Нет | Одна формула для всех |
| 6 | Быстрый старт | ✅ Да | ⚪ Нет | Одна формула для всех |
| 7 | Шаблоны | ✅ Да | ✅ Да | Выбираешь шаблоны по сфере |
| 8 | Яркий CTA | ✅ Да | ✅ Да | Адаптируешь под сферу |
| 9 | FAQ | ✅ Да | ✅✅ Очень | Вопросы подбираешь по сфере |

## 🎯 Примеры полных лендингов по сфере

### ИТ-команда

```
1. Hero: "ИТ-команды выбирают Kaiten"
2. Кейсы: 4-5 кейсов из IT (Яндекс, Сбербанк, etc)
3. Решения: 
   - Весь спринт видно на одной доске
   - Техдоки рядом с задачами
   - История всех изменений в одном месте
4. Гибкие блоки: Документы, Примеры команд, Интеграции, Графики
5. Перенос данных: Из Jira, GitHub, Azure DevOps
6. Быстрый старт: День 1-3
7. Шаблоны: Спринт, техдок, онбординг
8. Яркий CTA: "Запустите спринты в Kaiten"
9. FAQ: 8 вопросов про CI/CD, GitHub интеграции, etc
```

### Редакция

```
1. Hero: "Редакции выбирают Kaiten"
2. Кейсы: 4-5 кейсов из медиа (Эксмо, РИА Новости, etc)
3. Решения:
   - Одна статья, один статус, нет путаницы
   - Все правки в одном месте
   - График публикаций видно всем
4. Гибкие блоки: Документы, Примеры команд, Чат & видео, Графики
5. Перенос данных: Из Google Docs, Notion, CMS
6. Быстрый старт: День 1-3
7. Шаблоны: Статья, редактура, календарь публикаций
8. Яркий CTA: "Запустите редакцию по новому"
9. FAQ: 6 вопросов про версии, отзывы редактора, клиентский доступ
```

### Веб-агентство

```
1. Hero: "Веб-агентства выбирают Kaiten"
2. Кейсы: 4-5 кейсов из агентств (Логик, Fusion, etc)
3. Решения:
   - Видишь нагрузку каждого разработчика
   - 20 проектов, один процесс
   - Клиент видит только свой проект
4. Гибкие блоки: Примеры команд, Графики, AI помощник, Интеграции
5. Перенос данных: Из Asana, Monday, Trello
6. Быстрый старт: День 1-3
7. Шаблоны: Проект, задачи клиента, запуск проекта
8. Яркий CTA: "Начните управлять проектами как профессионалы"
9. FAQ: 6 вопросов про белую этикетку, клиентский доступ, интеграции Figma
```

### Юридический отдел

```
1. Hero: "Юридические отделы выбирают Kaiten"
2. Кейсы: 4-5 кейсов из юр/финанса (Сбербанк, Альфа-банк, etc)
3. Решения:
   - Все договоры на одной доске видно все статусы
   - Контроль над согласованиями между людьми
   - Автоматические напоминания о сроках
4. Гибкие блоки: Документы, Чат & видео, Примеры команд
5. Перенос данных: Из Excel, CMS, папок
6. Быстрый старт: День 1-3
7. Шаблоны: Договор, согласование, контроль сроков
8. Яркий CTA: "Полный контроль над договорами"
9. FAQ: 7 вопросов про безопасность, On-premise, интеграции с ЭП, ограничение доступа
```

## 🔄 Ключевой момент: от анализа болей к лендингу

### ЧТО ДЕЛАЕМ ДО НАПИСАНИЯ (внутренний анализ):

```
Исследуем целевую аудиторию ИТ-команд:

Боль 1: "Спринт горит, никто не знает где узкие места"
  → Решение Kaiten: Вся доска видна, видны зависимости

Боль 2: "Техдоки разлетаются по разным местам"
  → Решение Kaiten: Всё рядом с задачами

Боль 3: "Новичок месяц разбирается как что работает"
  → Решение Kaiten: Вся информация в одном месте
```

### ЧТО ПИШЕМ НА ЛЕНДИНГЕ (только решение):

```
❌ НЕПРАВИЛЬНО:
"Спринты часто горят и никто не знает где узкие места"
"Техдоки разлетаются по Confluence, Notion, Google Drive"
"Новичок месяц разбирается в процессах"

✅ ПРАВИЛЬНО:
"Весь спринт видно на одной доске. Видны все зависимости. Узких мест нет."
"Техдоки всегда рядом с задачей. Откроешь карточку — вся информация внутри."
"Новичок разбирается за день. Вся информация в одном месте."
```

## 🎬 Пошаговый процесс создания

```
ДЕНЬ 1: ИССЛЕДОВАНИЕ
├─ Определи аудиторию
├─ Найди их боли (анализ)
├─ Подбери кейсы
└─ Изучи фичи Kaiten

ДЕНЬ 2: ПЛАНИРОВАНИЕ СТРУКТУРЫ
├─ Реши какие гибкие блоки использовать
├─ Напиши решения (НЕ боли!)
└─ Составь список вопросов для FAQ

ДЕНЬ 2-3: НАПИСАНИЕ
├─ Hero блок
├─ Кейсы (форматирование)
├─ 3-4 решения (конкретно, без описания болей)
├─ 3-4 гибких блока
├─ Перенос данных
├─ Быстрый старт
├─ Шаблоны
├─ Яркий CTA
└─ FAQ

ДЕНЬ 3: ПРОВЕРКА И ПУБЛИКАЦИЯ
├─ SEO (title, description, keywords)
├─ Все ссылки работают
├─ Аналитика установлена
└─ ✅ Готово!
```

## 💡 Главный принцип

**Боли анализируем, чтобы лучше написать про решение.**  
**На лендинге показываем только решение и результат.**  
**Никогда не описываем текущие проблемы клиента.**

Человек уже знает свои проблемы. Ему нужно видеть как их решить.
