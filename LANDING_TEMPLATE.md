## 3. БЛОК ЗАКРЫТИЯ БОЛИ (3-4 боли)

**Повтори эту структуру для каждой боли:**

```
## [НАЗВАНИЕ БОЛИ В СЛОВАХ АУДИТОРИИ]

### Сейчас
[КОНКРЕТНОЕ ОПИСАНИЕ ТЕКУЩЕГО СОСТОЯНИЯ]
- [ПРИМЕР 1]
- [ПРИМЕР 2]
- [РЕЗУЛЬТАТ: ЧТО ЭТО ПРИВОДИТ]

### В Kaiten
[КАК ЭТО РЕШАЕТСЯ В KAITEN, ШАГ ЗА ШАГОМ]
- [ШАГ 1]
- [ШАГ 2]
- [РЕЗУЛЬТАТ: ЧТО ПОЛУЧАЕТСЯ]
```

**Пример для веб-агентства:**

```
## Не видно нагрузку на разработчиков

### Сейчас
20 проектов, разработчики скачут между ними.
Не видно, кто перегружен, кто может взять новую работу.
В итоге один человек работает на 150%, другой на 50%.

### В Kaiten
Видишь каждого разработчика и в каких проектах он работает.
Видишь сколько задач в работе у каждого.
Быстро находишь того, кто может взять новый проект.
Распределение справедливое, все загружены одинаково.
```

## 4. АКЦЕНТНЫЕ БЛОКИ С ФИЧАМИ (5-6 фич)

**Структура для каждой фичи:**

```
## [НАЗВАНИЕ ФИЧИ НА ЯЗЫКЕ АУДИТОРИИ, НЕ "ФИЧА"]

[ОПИСАНИЕ: ЧТО ПОЛУЧИШЬ, ПОЧЕМУ ВАЖНО, КАК ИСПОЛЬЗОВАТЬ]
```

**❌ НЕПРАВИЛЬНО:**
```
## Канбан доска
Визуализация процессов на доске с колонками статусов.
```

**✅ ПРАВИЛЬНО:**
```
## Вся работа видна на одной доске

Открыл Kaiten — видишь все задачи: какие в работе, какие ждут, какие готовы.
Сразу видно узкие места. Например, всё скопилось в колонке "QA" — значит, нужна помощь QA-инженеру.
Можешь оперативно помочь, проблему решить за часы, а не за дни.
```

**Примеры фич для разных сфер:**

**ИТ-команда:**
```
## Техдоки рядом с задачами
Когда разработчик берёт задачу, видит все документы прямо в карточке.
Не нужно прыгать между Confluence, Jira и Google Drive.
Новичок быстро разбирается, опытный видит всё актуальное, никто ничего не забывает.
```

**Редакция:**
```
## Одна статья, один статус, нет путаницы
Статья не может быть в двух местах одновременно.
Она либо на написание, либо на редактуру, либо на корректуру, либо готова.
Когда редактор видит статью, он знает ровно что ему делать. Нет путаницы с версиями.
```

**Веб-агентство:**
```
## Клиент видит только свой проект
Клиент заходит в Kaiten и видит ровно его работу.
Не видит других проектов и других клиентов.
При этом ты видишь все 20 проектов и можешь справедливо распределять ресурсы.
```

## 7. SEO И МЕТАДАННЫЕ

**Заполни эти поля:**

| Параметр | Значение |
|----------|----------|
| **Title** (50-60 символов) | [НАЗВАНИЕ ЛЕНДИНГА] |
| **Meta description** (150-160 символов) | [КРАТКОЕ ОПИСАНИЕ] |
| **Keywords** | [КЛЮЧ1], [КЛЮЧ2], [КЛЮЧ3], [КЛЮЧ4], [КЛЮЧ5] |
| **URL** | `kaiten.ru/teams/kaiten-for-[СФЕРА]` |

**Пример для редакции:**
```
Title: "Kaiten для редакций — управление статьями и публикациями"
Description: "Контроль над статьями от замысла до публикации. Видимость для всей редакции, автоматические напоминания о дедлайнах."
Keywords: "редакция, управление статьями, календарь публикаций, планирование статей"
URL: "kaiten.ru/teams/kaiten-for-edit"
```

## 8. ФИНАЛЬНЫЙ ЧЕК-ЛИСТ ПЕРЕД ПУБЛИКАЦИЕЙ

**Содержание:**
- [ ] Hero блок ясный и понятный
- [ ] 4-5 реальных кейсов из этой же сферы
- [ ] 3-4 боли с конкретным закрытием в Kaiten
- [ ] 5-6 фич на языке аудитории (не "фичи", а результаты)
- [ ] Шаблоны для скачивания
- [ ] Финальный CTA блок

**Язык:**
- [ ] Нет маркетингового лозунга ("революционный", "трансформирует")
- [ ] Нет пустых обещаний (не обещаем чудо)
- [ ] Нет корпоративного языка ("синергия", "оптимизировать ресурсы")
- [ ] Всё конкретно (есть примеры из жизни)
- [ ] Тон уверенный, но не дерзкий

**Технически:**
- [ ] Все ссылки на кейсы работают
- [ ] Логотипы кейсов хорошего качества
- [ ] Цитаты точные и проверены
- [ ] SEO заполнено (title, description, keywords, URL)
- [ ] Аналитика установлена (Яндекс.Метрика на кнопки)
- [ ] Страница быстро грузится
- [ ] Работает на мобильных

## 🎯 Метрика успеха

Лендинг удался, если человек из целевой сферы:

1. ✅ Читает заголовок — сразу понимает, это про него
2. ✅ Видит кейсы — думает "это работает с компаниями как моя"
3. ✅ Читает про боли — узнаёт свои трудности
4. ✅ Видит как Kaiten решает — думает "это именно то!"
5. ✅ Нажимает "Попробовать бесплатно" — хочет тестировать

**Если хотя бы один пункт не работает — переделай!**
