## 3. БЛОК ЗАКРЫТИЯ БОЛИ (3-4 боли)

⚠️ **ВАЖНО:** Мы НЕ описываем текущие боли! Показываем только РЕШЕНИЕ и РЕЗУЛЬТАТ.

**Повтори эту структуру для каждой боли:**

```
## [РЕЗУЛЬТАТ / ВЫГОДА В СЛОВАХ АУДИТОРИИ]

[КАК ЭТО РАБОТАЕТ В KAITEN, КОНКРЕТНЫЕ ШАГИ]
- [ШАГ 1]
- [ШАГ 2]
[РЕЗУЛЬТАТ И ВЫГОДА]
```

**Пример для веб-агентства:**

```
## Видишь нагрузку каждого разработчика

В Kaiten видишь каждого разработчика и в каких проектах он работает.
- Видишь сколько задач в работе у каждого
- Видишь, кто может взять новый проект
- Видишь где узкие места
Распределение справедливое, все загружены одинаково.
```

**Пример для редакции:**

```
## Одна статья, один статус, нет путаницы

В Kaiten каждая статья имеет ясный статус:
- "На написание" — автор пишет
- "На редактуру" — редактор проверяет
- "На корректуру" — корректор исправляет
- "Готова к публикации" — публикуется
Все правки в одном документе, одна версия финальная.
```

## 4. ГИБКИЕ БЛОКИ ПО СФЕРАМ (выбери 3-4 из них)

Это разные блоки в зависимости от сферы. Выбираешь какие подходят:

### Вариант 1: Документы и редактор Kaiten

```
## Встроенный редактор документов

В Kaiten есть собственный редактор документов.
Пиши статьи, регламенты, инструкции прямо в Kaiten.
Все видят историю правок, кто что менял, когда.
Не нужно скачивать Word, не нужно договариваться о версиях.
```

### Вариант 2: Примеры пространств и команд

```
## Каждая команда работает по-своему

- **Разработчики** — в спринтах, планируют на 2 недели
- **Редакция** — по календарю статей, планируют публикации
- **Маркетинг** — по кампаниям, каждая на месяц
- **Поддержка** — по очереди обращений, решают сразу

Все в одной системе, каждый видит только свое.
```

### Вариант 3: Чат и видеозвонки

```
## Общайтесь прямо в Kaiten

Встроенный чат для быстрых вопросов.
Видеозвонки прямо из карточки задачи.
Не нужно открывать Slack, Teams, Zoom — всё в одном месте.
История разговоров сохраняется вместе с задачей.
```

### Вариант 4: Графики и аналитика

```
## Видите прогресс в реальном времени

Диаграммы сгорания, графики завершения, метрики нагрузки.
Видишь, быстро ли вы движетесь к цели.
Видишь узкие места и перегруженные люди.
Принимаешь решения на основе данных, не на ощущениях.
```

### Вариант 5: Автоматизация и AI-помощник

```
## AI-помощник в Kaiten

Написал письмо по почте? AI преобразует в заметку в Kaiten.
Ставит теги, определяет сроки, предлагает исполнителя.
Рутинные задачи создаются полуавтоматически.
```

### Вариант 6: Декомпозиция задач

```
## Разбирайте сложные задачи на подзадачи

Большой проект — раскладываешь на этапы.
Каждый этап — на задачи.
Каждая задача — на подзадачи.
Видно зависимости, критический путь, где можно ускориться.
```

**Язык:**
- [ ] Нет маркетингового лозунга ("революционный", "трансформирует")

- [ ] Нет корпоративного языка ("синергия", "оптимизировать ресурсы")
- [ ] Всё конкретно (есть примеры из жизни)
- [ ] Тон уверенный, но не дерзкий

**Технически:**
- [ ] Все ссылки на кейсы работают
- [ ] Логотипы кейсов хорошего качества
- [ ] Цитаты точные и проверены
- [ ] SEO заполнено (title, description, keywords, URL)
- [ ] Аналитика установлена (Яндекс.Метрика на кнопки)
