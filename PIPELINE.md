# Kaiten Landing Factory Пайплайн

Этот документ описывает пошаговый процесс создания лендинга от идеи до публикации.

## 🎯 Идеальный сценарий

```
Ты: «Создай лендинг Kaiten для юридических отделов»
     ↓
Claude: Создаёт один промпт → генерирует готовый лендинг
     ↓
Разработчик: Берёт готовый HTML и публикует (БЕЗ правок в текстах)
```

## 📝 Как правильно задать задачу Claude

### Минимальный вариант (для самых простых лендингов)

```
Создай лендинг Kaiten для [СФЕРА/АУДИТОРИЯ]
```

Пример:
```
Создай лендинг Kaiten для юридических отделов
```

### Полный вариант (для более сложных лендингов)

```
Создай лендинг Kaiten для [СФЕРА/АУДИТОРИЯ]

Целевая аудитория: [описание кто именно]
Приоритетные фичи: [список 2-3 главных фич]
Известные конкуренты: [если есть]
Специальные требования: [если есть]
```

Пример:
```
Создай лендинг Kaiten для команд веб-агентств

Целевая аудитория: 
- Проект-менеджеры в веб-агентствах (5-50 человек)
- Те, кто управляет несколькими проектами для разных клиентов одновременно

Приоритетные фичи:
- Управление несколькими проектами одновременно
- Видимость для клиентов (кто может видеть, что происходит)
- Интеграция с популярными инструментами (Slack, Figma, GitHub)

Известные конкуренты:
- Asana
- Monday.com
- Jira

Специальные требования:
- Подчеркнуть, что это дешевле чем Asana/Monday
- Добавить кейс с Яндекс.Маркета или подобной крупной компанией
```

## 🔍 Что Claude должен проверить перед отправкой

Claude ДОЛЖЕН прочитать перед началом работы:

1. **kaiten-landing-factory/SKILL.md** — полное руководство
2. **kaiten-landing-factory/references/brand_policy/** — как писать про Kaiten
3. **kaiten-landing-factory/references/landing_tz_examples/** — примеры лендингов
4. **kaiten-landing-factory/references/links/** — ссылки на кейсы
5. **CLAUDE.md** — правила для конкретно этого проекта

## 📦 Что ты получишь на выходе

После одного промпта Claude создаст для тебя:

### Структура выходного файла

```
outputs/kaiten-for-[sfera]/
├── landing.html            # Готовая верстка (открыть в браузере)
├── landing.md              # Тексты в markdown (для архива)
├── seo.json               # SEO-данные
├── checklist.md           # Контрольный список перед публикацией
└── references.md          # Использованные источники
```

### Содержимое landing.html

- ✅ Полностью адаптивная верстка (mobile-first)
- ✅ Семантический HTML5
- ✅ Встроенный CSS (не нужны отдельные файлы)
- ✅ Готовые тексты без placeholder'ов
- ✅ Места под Яндекс.Метрику (закомментированы)
- ✅ Schema.org структурированные данные
- ✅ Alt-тексты для всех изображений

### Содержимое seo.json

```json
{
  "slug": "kaiten-dlya-yuridicheskikh-otdelov",
  "title": "Kaiten для юридических отделов — управление договорами и сроками",
  "description": "Сервис управления проектами для юридических отделов. Контроль над договорами, сроками и согласованиями.",
  "keywords": ["юридический отдел", "управление договорами", "контроль сроков"]
}
```

### Содержимое checklist.md

Контрольный список для финальной проверки перед публикацией:

- [ ] Тексты в стиле Kaiten (прочитал brand_policy)
- [ ] Все англицизмы объяснены
- [ ] Минимум 2 реальных кейса
- [ ] SEO-данные заполнены
- [ ] HTML адаптивен (проверил на мобильном)
- [ ] CTA-кнопки ведут в правильные места
- [ ] Alt-тексты на всех картинках
- [ ] Орфография и пунктуация ОК
- [ ] Тон соответствует голосу Kaiten

## 🔄 Если нужны правки

Если нужно что-то изменить:

### Простые правки (текст, SEO, CTA)

```
Измени в лендинге:
- Заголовок с [было] на [новое]
- Добавь кейс про [компанию]
- Переделай FAQ блок на вопросы про [тему]
```

### Структурные правки (добавить секцию, убрать блок)

```
Перестрой лендинг:
- Убери FAQ блок
- Добавь после фич секцию про интеграции
- Перемести кейсы выше
```

### Важно!

Когда просишь правки — Claude ПЕРЕПИСЫВАЕТ ТОЛЬКО нужный блок, всё остальное остаётся неизменным.

## 🎯 Критерии успеха

Лендинг готов к публикации, если:

- ✅ Все тексты написаны в стиле Kaiten (не помпезно, не скучно)
- ✅ Нет вымышленных кейсов (только реальные компании)
- ✅ Рекомендуемое количество контента:
  - Hero: 1-2 предложения
  - Секции: 2-3 абзаца каждая
  - Фичи: 1-2 предложения
  - Кейсы: 3-4 предложения
  - FAQ: 2-3 предложения на вопрос
- ✅ HTML валидный и проверен в браузере
- ✅ SEO-данные не превышают лимит символов
- ✅ Разработчик может взять HTML и публиковать БЕЗ правок
