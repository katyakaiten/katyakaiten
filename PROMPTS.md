# Готовые промпты для создания лендингов

Используй эти шаблоны как основу для быстрого старта.

## 📋 Основной промпт (универсальный)

Копируй этот промпт и подставляй свои значения:

```
Создай полный лендинг Kaiten для [СФЕРА]

Целевая аудитория:
[Опиши кто именно будет читать этот лендинг]

Ключевые проблемы этой аудитории:
- [Проблема 1]
- [Проблема 2]
- [Проблема 3]

Приоритетные фичи Kaiten для них:
- [Фича 1]
- [Фича 2]
- [Фича 3]

Специальные требования:
[Если есть особенности, которые нужно учесть]

Лендинг должен включить кейсы компаний:
[Компании, которые используют или могли бы использовать Kaiten]

На выходе жду:
1. Готовую HTML-верстку
2. SEO-данные (title, description, keywords)
3. Исследование аудитории (2-3 абзаца)
4. Чек-лист самопроверки
```

## 🎯 Примеры готовых промптов

### Пример 1: Для юридических отделов

```
Создай полный лендинг Kaiten для юридических отделов

Целевая аудитория:
Начальники юридических отделов и юристы крупных корпораций (50+ сотрудников). 
Они управляют договорами, согласованиями, сроками и работают с множеством заинтересованных сторон.

Ключевые проблемы этой аудитории:
- Сложно отследить статус всех договоров и согласований
- Много участников в процессе, не всегда понимают, кто на каком этапе
- Сроки по договорам легко пропустить
- Нет единого места для документов и коммуникации

Приоритетные фичи Kaiten для них:
- Управление статусами (черновик → согласование → подпись → архив)
- Видимость для всех участников
- Уведомления о сроках и изменениях
- Интеграция с корпоративными системами

На выходе жду:
1. Готовую HTML-верстку
2. SEO-данные (title: 50-60 символов, description, keywords)
3. Исследование целевой аудитории
4. Чек-лист перед публикацией
```

### Пример 2: Для редакций и медиа-компаний

```
Создай полный лендинг Kaiten для редакций и медиа-компаний

Целевая аудитория:
Главные редакторы, издатели и медиа-менеджеры, которые управляют множеством статей, авторов и публикаций одновременно.
Компании вроде Эксмо, ТАСС, Коммерсантъ.

Ключевые проблемы:
- Много авторов пишут одновременно, сложно отследить кто в каком этапе
- Сроки публикации критичны, нельзя пропустить
- Нужна обратная связь между редактором, корректором и автором
- Каждая статья проходит множество согласований

Приоритетные фичи:
- Управление статусами (идея → написание → редактура → корректура → публикация)
- Видимость редакторского потока для всей команды
- Комментарии и обсуждения на каждой статье
- История версий и изменений

На выходе жду:
1. Готовую HTML-верстку
2. SEO-данные
3. Кейс про Эксмо или подобную компанию (если знаю)
4. Чек-лист
```

### Пример 3: Для строительства и инжиниринга

```
Создай полный лендинг Kaiten для строительных компаний и инженерных проектов

Целевая аудитория:
Менеджеры проектов, прорабы и супервайзеры строительных компаний.
Управляют множеством этапов строительства, рабочей силой, поставками материалов и согласованиями.

Ключевые проблемы:
- Множество параллельных процессов на строительной площадке
- Зависимости между этапами — если один задержится, все страдают
- Нужна видимость для команды на объекте и в офисе
- Контроль сроков и бюджета

Приоритетные фичи:
- Управление этапами строительства
- Видимость статуса на каждом этапе
- Быстрые уведомления о проблемах
- Отчёты о прогрессе

На выходе жду:
1. Готовую HTML-верстку
2. SEO-данные
3. Кейс про строительную компанию или производство
4. Чек-лист
```

### Пример 4: Для веб-агентств

```
Создай полный лендинг Kaiten для веб-агентств

Целевая аудитория:
Проект-менеджеры и собственники веб-агентств (10-100 человек).
Управляют множеством проектов для разных клиентов одновременно.
Клиенты хотят видеть прогресс, но агентство не хочет открывать все данные.

Ключевые проблемы:
- Много проектов одновременно для разных клиентов
- Нужна видимость для клиентов (но ограниченная)
- Сложно контролировать бюджет и время на каждый проект
- Интеграции с инструментами: GitHub, Figma, Slack

Приоритетные фичи:
- Отдельные доски для каждого проекта
- Ограниченный доступ для клиентов
- Время-трекинг
- Интеграции с инструментами разработки

На выходе жду:
1. Готовую HTML-верстку
2. SEO-данные
3. Кейс про веб-агентство (если знаю)
4. Чек-лист
```

## 🔧 Промпты для правок

Если нужно что-то изменить:

### Простой текстовый апдейт

```
Измени в лендинге:
- Заголовок hero блока с "[было]" на "[новое]"
- В FAQ добавь вопрос про "[тему]"
- Измени СTA текст с "[было]" на "[новое]"
```

### Добавить кейс

```
Добавь в лендинг кейс:

Компания: [название]
Сфера: [индустрия]
Проблема была: [что мешало]
Результат: [что получилось]
Цифра результата: [метрика, если есть]

Остальной лендинг не трогай.
```

### Переструктурировать

```
Переделай структуру лендинга:
1. Оставь Hero блок как есть
2. Убери FAQ блок
3. После фич добавь блок "Интеграции" с [описание]
4. Остальное остаётся без изменений
```

## 💡 Советы по использованию промптов

### Когда использовать основной промпт?

- Создание нового лендинга с нуля
- Лендинг для новой целевой аудитории
- Полная переработка существующего лендинга

### Когда использовать примеры?

- Быстро адаптировать под похожую аудиторию
- Когда структура понятна, нужно только подставить свои данные

### Когда использовать промпты для правок?

- После первой версии, если нужны небольшие изменения
- Добавление новых кейсов или фич
- Изменение структуры без полной переработки
