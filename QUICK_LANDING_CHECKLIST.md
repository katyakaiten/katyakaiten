# 🚀 Быстрый чек-лист: создание лендинга за 3 дня

Используй этот документ как шпаргалку при создании нового лендинга для Kaiten.

---

## День 1: Исследование

### 1️⃣ Определи аудиторию (30 минут)

**Вопросы себе:**
- [ ] Кто руководитель этой команды?
- [ ] Какой размер команды?
- [ ] Какие у них основные процессы?

**Результат:** 2-3 предложения описания аудитории

### 2️⃣ Найди боли (1-2 часа)

**Используй эти источники:**
- [ ] Открытые отзывы и форумы (что жалуются люди)
- [ ] Сайты конкурентов (на что они делают акцент)
- [ ] LinkedIn и профессиональные сообщества
- [ ] Если есть — интервью с людьми из этой сферы

**Результат:** 3-4 основные боли в словах аудитории (не маркетолога)

### 3️⃣ Подбери кейсы (1-2 часа)

**Где:** https://kaiten.ru/blog/tag/case/

**Критерии:**
- [ ] Компания из той же сферы
- [ ] Есть конкретная цифра результата
- [ ] Есть цитата руководителя

**Результат:** 4-5 кейсов с цитатами и ссылками

### 4️⃣ Изучи фичи (1-2 часа)

**Где:** https://faq-ru.kaiten.site/

**Что выписываешь:**
- [ ] Какие фичи Kaiten решают каждую боль?
- [ ] Как они конкретно работают?
- [ ] Есть ли примеры использования?

**Результат:** 5-6 фич на языке аудитории

---

## День 2: Планирование и написание

### 5️⃣ Структура (30 минут)

**Каждый лендинг всегда:**

1. Hero блок (заголовок + подзаголовок + CTA)
2. Кейсы (4-5 штук)
3. Боли и их решение (3-4 боли)
4. Фичи Kaiten (5-6 фич)
5. Шаблоны для скачивания
6. Финальный CTA

**Запиши для себя:**
- [ ] Hero заголовок
- [ ] Hero подзаголовок
- [ ] 3-4 основные боли
- [ ] 5-6 фич (на языке аудитории, не маркетолога)
- [ ] Какие шаблоны дать для скачивания

### 6️⃣ Написание (4-6 часов)

**Блок за блоком:**

**Hero:**
- [ ] Заголовок: "[Сфера] выбирают Kaiten"
- [ ] Подзаголовок: 2 предложения про выгоду
- [ ] CTA: "Попробовать бесплатно"

**Кейсы:**
- [ ] Логотип компании (4-5 кейсов)
- [ ] Цитата с результатом
- [ ] Фамилия, должность, компания
- [ ] Ссылка на полный кейс

**Блок боли:**
- [ ] Название боли (в словах аудитории)
- [ ] Как это выглядит сейчас (конкретные примеры)
- [ ] Как это работает в Kaiten (шаг за шагом)
- [ ] Результат (если есть цифра)

**Фичи:**
- [ ] НЕ пишешь "это фича" — пишешь "вот что ты получишь"
- [ ] Конкретные примеры из жизни этой сферы
- [ ] Почему это важно именно для них

**Проверка:**
- [ ] Нет маркетингового лозунга
- [ ] Нет пустых обещаний
- [ ] Язык простой и понятный
- [ ] Тон уверенный, но не дерзкий

---

## День 3: SEO, проверка, публикация

### 7️⃣ SEO (30 минут)

**Заполни:**
- [ ] Title (50-60 символов с ключевым словом)
- [ ] Meta description (150-160 символов)
- [ ] Keywords (3-5 основных)
- [ ] URL (понятный, латиница или русский)

**Пример:**
```
Title: "Kaiten для редакций — управление статьями"
Description: "Контроль над статьями от замысла до публикации. Видимость для редакции."
Keywords: "редакция, статьи, планирование публикаций"
URL: "kaiten.ru/teams/kaiten-for-edit"
```

### 8️⃣ Финальная проверка (1 час)

**Язык и содержание:**
- [ ] Нет орфографических ошибок
- [ ] Нет маркетингового языка
- [ ] Есть реальные кейсы (не вымышленные)
- [ ] Все ссылки работают
- [ ] Логотипы кейсов хорошего качества
- [ ] Цитаты точные

**Технически:**
- [ ] Страница быстро грузится
- [ ] Работает на мобильных
- [ ] Аналитика установлена (Яндекс.Метрика)
- [ ] CTA кнопка ведёт в правильное место

### 9️⃣ Публикация (30 минут)

- [ ] Скопировал HTML код
- [ ] Вставил логотипы кейсов
- [ ] Установил Яндекс.Метрику на кнопки
- [ ] Проверил еще раз на продакшене
- [ ] Готово! 🎉

## 🚫 Главные ошибки (не делай!)

❌ **Маркетингово-говор:**
- "Революционное решение"
- "Полностью трансформирует"
- "Лучшее в своём классе"

**Вместо этого:**
- "Видишь все задачи на одной доске"
- "Экономишь час в день на поиск информации"
- "Люди перестали терять важные документы"

---

❌ **Описание фич вместо результатов:**
- "Система с канбан-доской"
- "Интеграции с популярными инструментами"
- "Расширенная система прав доступа"

**Вместо этого:**
- "Все задачи видны на одной доске — никто не застревает"
- "Данные автоматически синхронизируются, не нужно копировать вручную"
- "Сотрудники видят только свои проекты, клиент видит только его работу"

---

❌ **Вымышленные кейсы:**
- Только реальные кейсы с https://kaiten.ru/blog/tag/case/
- Проверяй фамилию и должность в оригинальном кейсе

---

## 📝 Шаблон текста для блока боли

**Копируй эту структуру для каждой боли:**

```
## [Название боли]

### Сейчас
[Конкретное описание того, как это выглядит сейчас]
- Пример 1
- Пример 2
- Результат: [что это приводит]

### В Kaiten
[Как это работает в Kaiten, шаг за шагом]
- Шаг 1
- Шаг 2
- Результат: [что получается в итоге]
```

**Пример:**

```
## Информация разлетается по разным местам

### Сейчас
Регламенты в Confluence, инструкции в Notion, чек-листы в Drive.
Новичок ищет информацию 2 часа, находит устаревшую версию, делает по-старому.

### В Kaiten
Открыл карточку задачи — вся информация здесь:
- Инструкция по выполнению
- Чек-лист проверки
- Регламент процесса
Новичок разбирается за 30 минут, делает правильно с первого раза.
```

## 🎯 Финальная метрика успеха

**Лендинг готов, если:**

1. ✅ Человек из целевой сферы читает заголовок — понимает, это для него
2. ✅ Он видит кейсы — "о, это работает с компаниями как моя"
3. ✅ Он видит как Kaiten решает его боли — "это именно то, что нам нужно"
4. ✅ Он видит фичи — "я сразу понимаю, как это работает"
5. ✅ Он нажимает "Попробовать бесплатно" — хочет тестировать

**Если хотя бы один пункт не выполнен — переделай!**
