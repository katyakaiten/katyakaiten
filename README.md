# Kaiten Landing Factory — Автоматизированный пайплайн создания лендингов

Полный пайплайн для генерации готовых к публикации лендинговых страниц Kaiten для разных сфер бизнеса, команд и рабочих процессов.

## 📋 Структура репозитория

```
kaiten-landing-factory/
├── SKILL.md                    # Основное руководство для агента
├── Kaiten_Landing_Factory_SKILL.docx  # Документ со скиллом
├── README.md                   # Описание скилла
├── references/
│   ├── brand_policy/           # Редполитика Kaiten
│   ├── landing_tz_examples/    # ТЗ-примеры лендингов для разных сфер
│   └── links/                  # Публичные ссылки на кейсы и базу знаний
│
├── outputs/                    # Готовые лендинги (генерируются)
│   ├── html/                   # Финальная верстка
│   ├── md/                     # Текстовые версии
│   └── specs/                  # ТЗ и спецификации
│
├── templates/                  # Шаблоны Kaiten и примеры
│   ├── html-template.html      # Базовый шаблон верстки
│   └── landing-schema.json     # JSON-схема для типизации
│
└── examples/                   # Примеры готовых лендингов
    ├── kaiten-for-kanban-teams/
    ├── kaiten-for-retail/
    ├── kaiten-for-construction/
    └── kaiten-for-media-outlets/
```

## 🚀 Быстрый старт

### Как создавать лендинги

**Идеальный промпт (одна команда для полного лендинга):**

```
Используй Kaiten Landing Factory Skill для создания лендинга.

Задача: создать лендинг Kaiten для [СФЕРА/АУДИТОРИЯ]

Исходные данные:
- Целевая аудитория: [описание]
- Приоритетные фичи: [список, если нужны]
- Конкурентные ссылки: [ссылки, если есть]
- Специальные требования: [если есть]

Выходные данные должны включать:
1. Исследование аудитории (1-2 абзаца)
2. Структура страницы (навигация по блокам)
3. Готовые тексты всех разделов
4. HTML-верстка с адаптивностью
5. SEO-данные (title, description, keywords)
6. CTA-кнопки и ссылки на кейсы
7. Чек-лист самопроверки

Опирайся на:
- kaiten-landing-factory/SKILL.md
- kaiten-landing-factory/references/brand_policy/
- kaiten-landing-factory/references/landing_tz_examples/
```

## 📊 Правила и стандарты

### Брендинг и язык
- Смотри `references/brand_policy/Редполитика+Kaiten.md`
- Тон: коллега, который разбирается в теме (умный, но не занудный)
- Язык: русский с объяснением термина (канбан, скрам — допускаются)
- Никаких англицизмов без раскрытия

### Структура лендинга
- Заголовок + подзаголовок
- Уникальное торговое предложение (УТП)
- Проблемы → Решение в Kaiten
- Ключевые фичи (3-5 основных)
- Кейсы из реальной жизни (Эксмо, METRO, Строительная компания и т.д.)
- Блок с компаниями, которые используют Kaiten
- FAQ
- CTA

### SEO и метаданные
- URL в нижнем регистре с дефисами: `kaiten-dlya-kanban-komand`
- Title: 50-60 символов
- Description: 150-160 символов
- Keywords: основной ключ + 3-5 вариантов

### Верстка
- Адаптивная (mobile-first)
- Semantic HTML5
- Стиль: Kaiten Design System
- Тёмная и светлая темы (если требует design system)

## 🔄 Workflow публикации

1. **Генерация** → Запрос лендинга с одним промптом (смотри выше)
2. **Проверка** → Я отдаю тебе:
   - Готовую верстку (HTML)
   - Тексты в Markdown
   - SEO-спецификацию
   - Чек-лист самопроверки
3. **Корректировки** → Только если нужны правки в текстах (БЕЗ переделки)
4. **Публикация** → Разработчик берет готовый HTML и публикует

## 🛠 Технический стек

- **Язык**: HTML5 + CSS3 (Flexbox/Grid)
- **Адаптивность**: Mobile-first responsive
- **Доступность**: WCAG 2.1 AA
- **Аналитика**: Места под Яндекс.Метрику и Google Analytics
- **SEO**: Schema.org структурированные данные
